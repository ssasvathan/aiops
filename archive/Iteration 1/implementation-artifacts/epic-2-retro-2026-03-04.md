## Deep Story Analysis (Stories 2-1 to 2-7)

### What Went Well

1. **Signal pipeline was built incrementally without breaking prior behavior.**
   - Story order was effective: collection -> detection -> peak -> sustained -> UNKNOWN propagation -> cache hardening -> outage safety.
2. **Safety semantics stayed explicit.**
   - UNKNOWN-not-zero, degraded-mode behavior, and deterministic caps were repeatedly reinforced.
3. **Review process surfaced real risks early enough to fix safely.**
   - High-severity findings were closed with targeted patches and regression tests.
4. **Test discipline improved as complexity increased.**
   - Later stories show stronger focused test suites for scheduler/evidence/integration boundaries.
5. **Architecture boundaries held under pressure.**
   - Shared primitives (health registry, structured logging, cache adapters, contracts) were reused instead of forked.

### Challenges and Friction Points

1. **Edge-case handling still tended to appear in review rather than pre-review.**
   - Examples: UNKNOWN-to-zero collapse, positional metric assumptions, outage classifier overreach, degraded cap enforcement gaps.
2. **Cross-stage coupling required careful regression protection.**
   - Scheduler wiring and stage output semantics created subtle downstream effects.
3. **Cache behavior required repeated hardening.**
   - Write-failure isolation, unknown-env TTL fallback, and compatibility key reads were necessary to avoid hidden regressions.
4. **Degraded-mode correctness demanded explicit branch coverage.**
   - Outage transitions and action-capping behavior needed dedicated verification paths.

### Pattern Synthesis

- **Recurring review feedback themes:**
  - Deterministic safety caps must be enforced at final decision assembly points.
  - Unknown/missing evidence semantics must be preserved through every stage.
  - Cache is optimization-only; failures must never become hot-path blockers.
- **Breakthrough moments:**
  - Story 2.3 established robust peak classification and cache scaffolding.
  - Story 2.5 made UNKNOWN propagation explicit end-to-end.
  - Story 2.7 introduced transition-based telemetry degradation handling and safe action posture.
- **Collaboration highlight:**
  - Adversarial review findings were handled without broad refactors, preserving delivery momentum and system stability.
